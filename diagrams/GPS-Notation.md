# GPS Visual Notation Standard

## Purpose
Define the visual language used for GPS architecture diagrams.

## Core Shapes

- Purpose: represents institutional intent
- Actor: represents participants and stakeholders
- Capability: represents durable institutional ability
- Resource: represents enabling assets
- Decision: represents commitments and choices
- Outcome: represents produced value or impact
- Knowledge: represents information and understanding
- Learning: represents adaptation mechanisms

## Relationship Lines

Supports: enables achievement
Requires: dependency
Creates: produces result
Consumes: uses resource
Owns: accountability
Measures: evaluation
Learns From: improvement source

## Diagram Rule
Every visual element should map to a registered GPS artifact.
