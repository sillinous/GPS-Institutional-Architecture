# GPS v0.2 Assessment Pipeline Design

## Purpose
Define the orchestration flow for GPS assessments.

## Pipeline

1. Load institutional model.
2. Validate model structure.
3. Store entities and relationships.
4. Execute assessment rules.
5. Generate findings.
6. Produce traceable recommendations.

## Principles

- Repeatability
- Explainability
- Traceability
- Extensibility
